关键技术

异步 ETL · BullMQ

concurrency=3 控制并发,lockDuration=5min + stalledInterval=1min 显式拉大,避免 Windows + onnxruntime 启动慢导致 stalled 误判。失败 attempts=3,指数退避 2s→4s→8s。

双轨制

ragTrack 路由

xlsx/xls/csv 走 SQL 轨 — 每行一个 chunk,metadata 携带 sheetName/columns/rowIndices,引用可精准到行级;其他格式走 VECTOR 轨 — 文本切片 + embedding。

检索增强

Summary + FAQ + Ctx

ETL 阶段为文档额外生成三类辅助 chunk:summary(整文档摘要)、faq(每段 3-5 个可能问题)、ctx(灰度,Anthropic Contextual Retrieval),召回率提升 49%-67%。

可观测性

Prometheus 埋点

ETL 完成时 recordEtlComplete 上报耗时;embedding/llm 熔断状态变化通过 onStateChange 上报 rag_circuit_breaker_state gauge,生产排障先看 /metrics。

可靠性

限流 + 熔断

所有 embedding 调用走 limitAndBreaker:p-limit(5) 限流 + opossum 熔断(错误率 > 50% 触发 30s 短路)。LLM 独立熔断,参数 p-limit(8)、timeout=60s。

中文兼容

文件名 latin1→utf8

multer 2.x 永远按 latin1 解析 multipart 头里的 UTF-8 字节流,controller + service 双层反向解码 Buffer.from(raw,'latin1').toString('utf8'),否则中文文件名存储即乱码。

检索增强

HyDE + Multi-Query

HyDE 让 LLM 生成"假设性回答"拼接检索 query,提升召回;ai.rag.p2.multiQuery 灰度开启多路改写,每路独立检索后本地 rrfFusion(k=60) 合并。

检索精排

BM25+Dense+RRF+Rerank

Qdrant 原生 hybrid(ai.rag.p1.hybrid 灰度)用 sparse prefetch + RRF;默认 dense-only。召回 20 → cross-encoder rerank 截到 4,rerank 不足时用 raw recall 顺序补。

幻觉防护

两道拒答闸

① hard check:query 与 sources 关键词 bigram 重叠率 <5% 且 token 数 ≥10 触发拒答;ai.rag.p3.crag 灰度开启 LLM 评估 IRRELEVANT 过滤。两道闸都阻断"LLM 看到无关内容强行编"。

成本控制

客户端断开即停

res.on('close') 触发 AbortController,LangChain stream 内部检测到 abort 立即 break;for await 循环内同时检查 signal.aborted 兜底。不让 LLM 继续生成用户看不到的 token。

多轮记忆

历史拼装 + 灰度检索

buildHistoryContext(6) 拉最近 6 轮 messages 直接喂 LLM;ai.rag.p1.historyInRetrieval 灰度开启时把上一轮用户问题拼接到检索 query,解决"上面那个制度的第三条"指代问题。

双轨引用

CitationDto 行级定位

SQL 轨道 metadata 携带 sheetName/columns/rowIndices,前端能精准定位到具体单元格;VECTOR 轨道仅 chunkIndex。LLM 生成时强制 [n] 角标引用,sanitizeCitationMarkers 兜底越界替换。
